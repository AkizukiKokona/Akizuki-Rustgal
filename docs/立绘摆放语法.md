# 立绘摆放语法

## 概述

Akizuki*Rustgal 的剧本语言支持精确控制立绘（角色图片）在屏幕上的位置与大小。所有位置均使用百分比坐标（0.0–1.0），确保在任何分辨率下布局一致，实现多分辨率自适应。

## 基本语法

立绘入场使用 `+` 指令，完整语法如下：

```
+ <角色名> [<立绘资源>] [<位置>] [at <x>[,<y>]] [size <倍数>] [with <过渡效果>]
```

### 参数说明

| 参数 | 说明 | 默认值 |
|------|------|--------|
| 角色名 | 角色的标识名称 | 必填 |
| 立绘资源 | 括号内指定立绘资源名，如 `(kokonabody1)` | 无 |
| 位置 | 预设位置：`居左` / `居中` / `居右`（或英文 `left` / `center` / `right`） | 居中 |
| `at x,y` | 精确百分比位置，x、y 均为 0.0–1.0 的小数 | 无（使用预设位置） |
| `size 倍数` | 立绘大小倍数，> 0 | 1.0 |
| `with 过渡` | 入场过渡效果，如 `fade`、`dissolve`、`slide_left` 等 | 视场景而定 |

## 位置详解

### 预设位置

预设位置有三种：

- **居左 / left**：水平 25% 位置
- **居中 / center**：水平 50% 位置
- **居右 / right**：水平 75% 位置

当场上有两个角色时，会自动调整为左-右布局。

### 精确百分比位置（`at` 语法）

使用 `at x,y` 可以精确指定立绘在屏幕上的位置：

```
+ 心夏 at 0.3,0.8
```

- **x**：水平位置百分比。0.0 = 屏幕最左，0.5 = 屏幕正中，1.0 = 屏幕最右
- **y**：垂直位置百分比。0.0 = 屏幕最顶，1.0 = 底部站立（立绘底部贴齐屏幕底部）
  - y = 1.0 是默认值，立绘底部对齐屏幕底部（标准立绘站姿）
  - y < 1.0 时，立绘中心点对齐到该百分比位置

也可以只指定 x：

```
+ 心夏 at 0.5
```

此时 y 使用默认值 1.0（底部站立）。

### 大小倍数（`size` 语法）

使用 `size 倍数` 可以调整立绘的整体大小：

```
+ 心夏 size 1.2
```

- 倍数 > 1.0：放大立绘
- 倍数 < 1.0：缩小立绘
- 立绘的自然高度为屏幕高度的 80%，乘以倍数后得到实际显示高度

### 组合使用

位置、大小可以与立绘资源、预设位置、过渡效果自由组合：

```
+ 心夏 (kokonabody1) at 0.3,0.8 size 1.2 with fade
+ 心夏 at 0.5 size 0.9 with dissolve
+ 心夏 (happy) 居左 size 1.1 with slide_left
```

### 出场语法

出场使用 `-` 指令，语法类似：

```
- <角色名> [with <过渡效果>]
```

## 默认值

当不指定 `at` 和 `size` 时：

- x = 0.5（水平居中）
- y = 1.0（底部站立）
- size = 1.0（原始大小）

这与传统视觉小说的立绘摆放习惯一致。

## 编辑器预览

在 Akizuki*Rustgal 编辑器中，切换到右栏「立绘」标签页可以实时预览立绘效果：

1. 从下拉列表中选择立绘资源（需放在 `assets/characters/` 目录下）
2. 拖动 X 位置、Y 位置、大小滑块调整参数
3. 预览区实时显示立绘摆放效果（16:9 比例，模拟游戏屏幕）
4. 点击「复制到剪贴板」或「追加到脚本」生成对应语法

## 多分辨率适配

由于所有位置与大小均基于百分比计算，游戏在不同分辨率下会保持一致的布局比例：

- 立绘始终按屏幕高度的 80% 作为自然高度基准
- 水平/垂直位置均按屏幕宽高百分比计算
- 在 1920×1080、1280×720、2560×1440 等分辨率下布局比例完全一致
